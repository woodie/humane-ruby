# humane v0.9.3

## One `Humane` module, `distance_in_time`/`time_ago` split

Still converging on the final API -- versioned `0.9.3`, not `1.0.0`, on
purpose. `Humane::TimeFormatter`/`Humane::SizeFormatter` are gone;
`.human_size`, `.distance_in_time`, and `.time_ago` move directly onto the
`Humane` module. The old two-argument `.time_ago` is renamed
`.distance_in_time`; a new one-argument `.time_ago` takes its place. Same
naming split as ActionView's own `distance_of_time_in_words` (explicit
two-input core) and `time_ago_in_words` (one-input convenience wrapping it
with `Time.now`) -- deliberately borrowed, not invented. `humane` (Go) and
`humane-swift` picked up the equivalent changes in the same session -- see
`docs/COWORK.md`'s `v0.9.3` entry for the full rationale, including why no
`include`-able mixin was added despite Ruby being able to support one
(checked against `scandalous`'s own history and found unnecessary).

```ruby
Humane::SizeFormatter.human_size(225_935)        # old
Humane.human_size(225_935)                       # new -- "226 KB"

Humane::TimeFormatter.time_ago(mtime, now)        # old -- "3 minutes ago"

Humane.distance_in_time(mtime, now)               # new -- explicit relative_to, same output
Humane.time_ago(mtime)                            # new -- one argument, relative_to is Time.now
```

- **No more `TimeFormatter`/`SizeFormatter` classes.** `.human_size`,
  `.distance_in_time`, and `.time_ago` are module methods on `Humane`
  directly.
- **`.distance_in_time`** is the explicit, fully-tested core -- unchanged
  behavior from the old `.time_ago`, just renamed and un-namespaced. This
  is what specs should call.
- **`.time_ago`** is a new one-argument convenience for the common "drop
  into a view" case. Supplies `Time.now` as `relative_to` internally; every
  keyword option (`approximate:`, `include_seconds:`, `when_nil:`) still
  applies.
- Not invented from scratch: `scandalous`'s own `web.rb` `helpers do` block
  was already hand-wrapping the old two-argument `.time_ago` just to
  supply `Time.now`, the same shape `lambada`'s Go template FuncMap and
  `zouk`'s `ScanEntry.timeAgo` had independently hand-built for themselves.
  The new one-argument `.time_ago` here (and `humane`/`humane-swift`'s
  matching convenience) lets all three drop their wrapper.

## Upgrading

Every call site needs a change:

```ruby
# old
Humane::SizeFormatter.human_size(size)
Humane::TimeFormatter.time_ago(t, now)

# new
Humane.human_size(size)
Humane.distance_in_time(t, now)
```

If the call site was only ever supplying `Time.now` for `relative_to` (the
common case in a Sinatra/Rails helper), simplify to the new one-argument
form instead:

```ruby
# old
def time_ago(time)
  Humane::TimeFormatter.time_ago(time, Time.now)
end

# new
def time_ago(time)
  Humane.time_ago(time)
end
```

`scandalous` is the only known Ruby consumer and needs this follow-up pass
-- see its own `docs/COWORK.md` once that happens.
