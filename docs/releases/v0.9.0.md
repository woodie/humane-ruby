# humane v0.9.0

## A full API rethink -- class methods, new defaults, nil-aware `time_ago`

Breaking change throughout. Through `v0.6.0` this gem mirrored
Foundation's own formatter-class API shape. With three real consumers now
shipped, the bar changed: feel as simple to drop into a Ruby HTML template
as ActionView's own helpers, while staying consistent with `humane` (Go)
and `humane-swift`. See `docs/COWORK.md`'s `v0.9.0` entry for the full
rationale. Versioned `0.9.0` (a jump from `v0.6.0`, matching the identical
jump `humane` and `humane-swift` make in the same session) rather than
`v0.7.0` or `v1.0.0` -- this redesign is the most considered this API has
ever been, informed by three real consumers rather than a first guess, but
`0.9.0` treats it as a release-candidate for `v1.0.0` rather than a full
stability promise just yet.

```ruby
Humane::SizeFormatter.new.string(from_byte_count: 225_935) # old
Humane::SizeFormatter.human_size(225_935)                  # new -- "226 KB"

Humane::TimeFormatter.new(approximate: true).string(at: t, relative_to: now) # old
Humane::TimeFormatter.time_ago(t, now)                                       # new -- "about 15 hours ago"
```

- **No more instantiation.** `SizeFormatter`/`TimeFormatter` are now class
  methods (`.human_size`, `.time_ago`) instead of instances built with
  `.new`. Configuration moves from constructor keyword arguments to
  call-site keyword arguments.
- **`human_size` rounds to 3 significant figures**, not 2 -- fixes
  `5_240_000_000` -> `"5.24 GB"` (previously `"5.2 GB"`), and bakes in two
  other real-hardware findings: `0` -> `"Zero KB"`, and byte-scale values
  spell out `"bytes"` (`"7 bytes"`) rather than using a `"B"` label. See
  `docs/COMMENTS.md` for the derivation.
- **`time_ago`'s `approximate:` now defaults to `true`**, matching
  ActionView's own always-on-past-the-hour-boundary behavior, and matching
  what `scandalous` already passed explicitly. `include_seconds:` stays
  `false` by default, unchanged.
- **`time_ago` accepts `at: nil`** and a new `when_nil:` keyword, returned
  as-is (no formatting) when `at` is `nil` -- lets a caller collapse a
  guard-then-fallback dance into one call.

## Upgrading

Every call site needs a change:

```ruby
# old
Humane::TimeFormatter.new.string(at: t, relative_to: now)

# new
Humane::TimeFormatter.time_ago(t, now)
```

If you were relying on `approximate: false` (the old default) for exact
wording rather than "about" phrasing, pass it explicitly now:

```ruby
Humane::TimeFormatter.time_ago(t, now, approximate: false)
```

`scandalous` is the only known Ruby consumer and needs this follow-up pass
-- see its own `docs/COWORK.md` once that happens.
